# Dataset Architecture & Pipeline Guide 

The original raw and processed data components for this repository are documented below. Due to storage constraints and size optimization practices, heavy data assets are excluded from version control tracking.

## Folder Topology

```text
.root/
└── Datasets/
    └── extracted_files/                                #retail_contaminated_dataset.zip
        ├── customer master.csv
        ├── inventory_snapshot.csv
        ├── promotions.csv
        ├── sales transactions.csv
        ├── sku_inventory_flags.csv
        ├── sku master.csv
        └── store master.csv

        └── processed_sales_transactions.parquet        #explicitly generated by checkpointing the sales transactions data after processing

```

**Original Dataset Official Link:** [Synthetic Retail Dataset - 10 Million Transactions](https://www.kaggle.com/datasets/mrayyanshehzad/synthetic-retail-dataset-10-million-transactions)

## Data Pipeline & Intermediate Checkpointing

* __Checkpoint Generation:__ The intermediate file `processed_sales_transactions.parquet` was generated at the conclusion of the execution pipeline inside __`03_Feature_Engineering.ipynb`__ [cite: x].
* **State Preservation:** This file encapsulates full feature scaling, engineering changes, and transforms via deliberate state checkpointing [cite: x].
* __Downstream Integration:__ This specific `.parquet` file acts as the primary execution input for the data analysis suite executed within __`04_EDA_Analysis.ipynb`__ [cite: x].
